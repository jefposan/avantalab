# Visão do produto — Vendas Mobile

## Público principal

O Vendas Mobile é pensado para vendedores que trabalham de forma prática, em campo, sem estrutura fiscal complexa.

Perfis típicos:

- vendedor autônomo;
- vendedor porta a porta;
- revendedor de cosméticos;
- representante informal;
- vendedor de catálogo;
- consultor independente;
- profissional que vende produtos de várias marcas.

## Problema

Esse vendedor normalmente precisa de algo simples:

- saber o que vendeu;
- saber para quem vendeu;
- saber quanto tem a receber;
- organizar clientes;
- consultar catálogo;
- registrar pedidos rapidamente;
- acompanhar resultados do mês;
- trocar de marca/produto sem recomeçar tudo.

Ele não precisa, pelo menos na primeira versão, de:

- nota fiscal;
- cupom fiscal;
- SPED;
- integração contábil;
- ERP completo.

## Proposta

Criar um PWA mobile simples e rápido para:

- carregar produtos por pacote ou planilha;
- cadastrar clientes;
- fazer vendas;
- registrar pagamentos ou situação da venda;
- gerar relatórios simples;
- trabalhar com múltiplas marcas.

## Fluxo principal

1. Usuário instala/ativa o módulo no AvantaLab.
2. Acessa a URL do PWA Vendas Mobile.
3. Escolhe carregar um pacote de produtos, por exemplo Tridium.
4. Opcionalmente importa produtos de outra marca por XLS/XLSX.
5. Cadastra clientes.
6. Registra vendas.
7. Consulta relatórios.
8. Acessa a gestão pelo botão para o AvantaLab `/mobile`.

## Primeira versão desejada

### Mobile/PWA

- login com conta AvantaLab;
- tela inicial com resumo do mês;
- lista de produtos;
- busca por produto;
- carrinho simples;
- cadastro rápido de cliente;
- registro de venda;
- histórico de vendas;
- relatório mensal;
- botão para abrir gestão AvantaLab.

### Gestão mínima

Inicialmente, a gestão pode ser simples e separada:

- produtos;
- clientes;
- vendas;
- relatórios.

A gestão web completa dentro do AvantaLab fica para fase futura.
